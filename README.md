# Atividade_BubbleSort

# Análise dos Resultados
O número de comparações cresce bastante conforme o tamanho do vetor aumenta.
Tanto o Bubble Sort quanto o Selection Sort possuem complexidade O(n²).
O Bubble Sort realiza muito mais movimentações quando o vetor está em ordem decrescente.
O Selection Sort faz praticamente o mesmo número de comparações que o Bubble Sort, mas realiza menos movimentações.

# Algoritmo com Melhor Desempenho
Mesmo ambos sendo O(n²), o Selection Sort apresentou melhor desempenho geral, principalmente porque ele realiza menos trocas e, por conta disso, o tempo de execução tende a ser menor para vetores grandes.
O Bubble Sort foi o mais lento, principalmente no vetor de 10000 posições, pois no pior caso ele realiza muitas trocas.
